# Mitwirken

Vielen Dank für dein Interesse an MatchPoint! ❤️

MatchPoint ist ein Open-Source-Projekt und lebt von der Community. Ganz gleich, ob du einen Fehler gefunden, eine Idee für eine neue Funktion oder selbst Code geschrieben hast – jede Form der Unterstützung ist willkommen.

## Wie kann ich helfen?

Es gibt viele Möglichkeiten, sich einzubringen:

- 🐞 Fehler melden
- 💡 Neue Funktionen vorschlagen
- 🌍 Übersetzungen verbessern
- 📖 Dokumentation erweitern
- 💻 Code beitragen
- 🧪 Neue Geräte testen und Erfahrungen teilen

Auch kleine Beiträge helfen dabei, MatchPoint weiter zu verbessern.

---

## Fehler melden

Falls dir ein Fehler auffällt, eröffne bitte ein GitHub Issue.

Hilfreich sind dabei möglichst viele Informationen, beispielsweise:

- verwendetes Betriebssystem
- Browser
- Gerät
- Schritte zum Nachstellen
- erwartetes Verhalten
- tatsächliches Verhalten

Screenshots oder Bildschirmaufnahmen erleichtern die Fehlersuche zusätzlich.

---

## Feature Requests

Neue Ideen sind jederzeit willkommen.

Vor größeren Änderungen empfiehlt es sich, zunächst ein GitHub Issue zu eröffnen. So können mögliche Lösungsansätze gemeinsam diskutiert werden.

---

## Entwicklungsumgebung

Eine Anleitung zum lokalen Einrichten des Projekts findest du im Kapitel **Entwicklungsumgebung**.

Kurz zusammengefasst:

```bash
git clone https://github.com/engel-b/matchpoint.git
cd matchpoint
npm install
npm run dev
```

---

## Coding Style

MatchPoint setzt auf eine einheitliche Codebasis.

Vor jedem Commit werden geänderte Dateien automatisch geprüft und formatiert.

Verwendet werden unter anderem:

- TypeScript
- ESLint
- Prettier
- Husky
- lint-staged

Bitte verzichte daher auf manuelle Formatierungsanpassungen, sofern sie nicht notwendig sind.

---

## Pull Requests

Pull Requests sollten möglichst klein und thematisch klar abgegrenzt sein.

Vor dem Erstellen eines Pull Requests sollten folgende Befehle erfolgreich ausgeführt werden:

```bash
npm run format:check
npm run lint
npm run build
npm run docs:build
```

Bitte beschreibe im Pull Request kurz:

- Was wurde geändert?
- Warum wurde die Änderung vorgenommen?
- Gibt es Besonderheiten oder Einschränkungen?

---

## Dokumentation

Neue Funktionen sollten nach Möglichkeit auch in der Dokumentation beschrieben werden.

Insbesondere betrifft dies:

- Benutzerhandbuch
- FAQ
- Entwicklerdokumentation
- Screenshots (falls erforderlich)

---

## Gerätekompatibilität

MatchPoint unterstützt zahlreiche externe Eingabegeräte.

Falls du ein bisher nicht getestetes Gerät erfolgreich verwendest, freuen wir uns über eine Ergänzung der Kompatibilitätsliste.

Interessant sind beispielsweise:

- Hersteller und Modell
- Verbindung (USB, Bluetooth oder 2,4 GHz)
- verwendete Tastencodes
- Besonderheiten

---

## Lizenz

Mit dem Einreichen eines Pull Requests erklärst du dich damit einverstanden, dass dein Beitrag unter der Projektlizenz veröffentlicht wird.
